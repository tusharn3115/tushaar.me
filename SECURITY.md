# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates:

| Version | Supported |
| ------- | --------- |
| 5.1.x   | ✅ Yes    |
| 5.0.x   | ❌ No     |
| 4.0.x   | ✅ Yes    |
| < 4.0   | ❌ No     |

Only the versions marked as **supported** will receive security patches.  
Please upgrade to a supported version to ensure you are protected.

---

## Reporting a Vulnerability

If you discover a security vulnerability, we appreciate your help in responsibly disclosing it.

### How to Report
- **Do not** open a public GitHub issue.
- Report vulnerabilities by emailing: **security@yourdomain.com**  
  *(Replace this with your actual security contact email.)*

### What to Include
Please include as much of the following information as possible:
- A clear description of the vulnerability
- Steps to reproduce the issue
- Affected versions
- Potential impact
- Any proof-of-concept code or screenshots (if applicable)

### Response Timeline
- You can expect an **initial response within 48 hours**
- We will keep you informed about the progress and resolution
- If the vulnerability is accepted, we will work on a fix and coordinate a responsible disclosure
- If the report is declined, we will explain why

### Disclosure Policy
Please allow us reasonable time to investigate and resolve the issue before any public disclosure.

---

Thank you for helping keep this project and its users secure.
