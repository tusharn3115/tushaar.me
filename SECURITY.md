# Security Policy

## Supported Versions

This project is a personal portfolio website. Security updates are applied to the latest version only.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly by following these steps:

1. **Do not** open a public issue for the vulnerability.
2. Email the maintainer directly at: [Report via GitHub Security Advisory](https://github.com/tusharn3115/tushaar.me/security/advisories/new)
3. Alternatively, you can open a private security advisory on GitHub.

### What to Include

When reporting a vulnerability, please include:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (if applicable)

### Response Timeline

- **Initial Response**: Within 48 hours of receiving the report
- **Status Updates**: Every 7 days until the issue is resolved
- **Resolution**: We aim to address critical vulnerabilities within 30 days

## Security Best Practices

This project follows security best practices including:

- Regular dependency updates to patch known vulnerabilities
- Code review for security considerations
- Secure deployment practices
- HTTPS-only connections in production
- Content Security Policy (CSP) headers where applicable

## Disclosure Policy

- Security issues are fixed privately before public disclosure
- After a fix is deployed, a security advisory will be published
- Credit will be given to security researchers who responsibly disclose vulnerabilities (unless they prefer to remain anonymous)

## Contact

For security-related questions or concerns, please contact:

- **GitHub Issues**: For general questions (non-security related)
- **Security Advisory**: For security vulnerabilities (preferred method)
- **Repository**: [https://github.com/tusharn3115/tushaar.me](https://github.com/tusharn3115/tushaar.me)

Thank you for helping keep this project secure!
